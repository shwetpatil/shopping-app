# Coding Standards

## TypeScript Guidelines

- Use strict mode
- Explicit types for function parameters
- Avoid `any` type
- Use interfaces for object shapes

## Naming Conventions

- Classes: PascalCase
- Functions: camelCase
- Constants: UPPER_SNAKE_CASE
- Files: kebab-case.ts

## Code Style

- Use Prettier for formatting
- ESLint for linting
- 2 spaces indentation
- Single quotes for strings
